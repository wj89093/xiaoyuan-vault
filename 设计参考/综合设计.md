## 核心定位

**10人团队级个人/团队知识库**，类Obsidian，AI原生设计。

目标：知识库新建后，和AI充分沟通后建设，目录随业务调整而弹性变化。

---

## 核心设计原则

### 1. 技能不依赖目录
```
技能只依赖：内容类型(type) + 用户意图
目录 = AI和用户协商出来的共识，可调整
```

### 2. 目录弹性
```
Phase 1：先用通用结构
和AI充分沟通后，AI建议新目录 → 用户确认后才建
目录调整 → AI自动更新所有 frontmatter.type
```

### 3. 双层页面积累知识
```
页面只增不减，时间线 append-only
每次更新是叠加，不是覆盖
```

---

## 三层架构

```
原始文件 (.raw/)  →  知识库（AI维护）  →  系统规范（schema + RESOLVER）
Immutable              Wiki                  驱动AI行为
```

---

## 目录结构（Phase 1 最小集）

```
知识库/
├── index.md          # AI自动维护的内容目录（按 type 聚合）
├── log.md           # 操作日志（append-only）
│
├── .raw/            # 原始文件（Immutable）
│
└── 0-收集/          # 快速入口，不分类，等AI和用户协商后迁移
```

**目录是协商出来的**，初始只有 0-收集/，AI根据内容建议新增目录。

## 双层页面结构

每页分上下两部分（以 `---` 分隔）：

### 上方：编译真相（当前状态）

```yaml
---
title: 公司名
type: company           # AI和用户协商确定，可调整
tags: [生物, 融资]
status: active          # active | archived
summary: 一句话摘要
created: 2026-01-01
updated: 2026-04-27
confidence: high        # high | medium | low
relationships:          # 自动抽取的关系
  - type: invested_in
    target: 红杉资本
    confidence: EXTRACTED
    source: 腾讯新闻
---

# 编译真相

## 基本信息
- 成立时间：
- 总部：
- 融资阶段：

## Open Threads（活跃事项）
- [ ] 待确认创始人背景

## See Also
- [[相关公司A]]
- [[投资机构B]]

---  <!-- 分界线，以下永不修改 -->

## 时间线（Append-only）
```

- 格式：`## [日期] 类型 | 内容`

## 技能系统

### 目录结构

```
skills/
├── RESOLVER.md          # 🔑 决策树：收到内容 → 路由到哪个技能
├── enrich.md             # 入库：判断内容类型 → 和用户协商放哪
├── query.md              # 查询：搜索 + 综合回答
├── ingest.md             # 导入：处理外部文件/链接
├── maintain.md           # 维护：定期检查 + 提醒
└── conventions/
    ├── quality.md        # 质量标准
    └── citation.md       # 引用格式
```

### RESOLVER.md（不写死目录）

```markdown
# RESOLVER - 内容路由决策树

收到内容后，判断：

1. 是用户提问？
   → 走 query.md

2. 是外部文件/链接？
   → 走 ingest.md → enrich.md

3. 是待处理的新内容？
   → 走 enrich.md

enrich 判断逻辑：
- 有人名 → 问用户"这是谁？属于哪类？"
- 有公司名 → 问用户"这是公司还是项目？"
- 有会议/时间 → 建议进 4-会议/
- 缺信息 → 进 0-收集/，等用户确认后迁移

⚠️ 不写死目录！每次遇到新类型，和用户协商创建新目录。
```

### Type 决定行为，不依赖目录

```typescript
// ❌ 旧：hardcode 目录
if (folder === '2-公司') { ... }

// ✅ 新：内容导向
if (frontmatter.type === 'company') { ... }
// type 是 enrich 时和用户协商确定的
// 目录变了，只要 type 不变，技能照常工作
```

## 搜索架构（从 GBrain 继承）

| 层级 | 功能 |
|------|------|
| L1 | SQLite FTS5 全文搜索（已有）|
| L2 | Typed links 关系搜索 |
| L3 | Embedding 语义搜索 |
| L4 | RRF 综合排名 |

```
用户查询 "合成生物学公司"
→ FTS5: 命中["合成生物学"]
→ Links: 相关公司页面
→ Embedding: 语义相似的"生物制造"相关页
→ RRF合并去重 → 返回排序结果
```

## 自动 Enrich Pipeline

```
新文件进入 0-收集/
    ↓
自动触发 enrich 技能
    ↓
AI判断内容类型 → 和用户协商"这是啥、放哪"
    ↓
用户确认 type
    ↓
创建/更新目标页面
    ↓
提取关系 + 更新 frontmatter
    ↓
更新 index.md 目录
    ↓
追加 log.md 时间线
    ↓
如需迁移 → AI自动移动文件 + 更新 index
```
