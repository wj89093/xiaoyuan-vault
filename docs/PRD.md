> 创建时间：2026-04-23
> 更新时间：2026-04-29
> 版本：v1.1
> 目标：5-10 人团队级知识库，类 Obsidian

---

## 一、产品定位

**一句话：** 面向 1-20 人团队的个人/团队知识库，类 Obsidian，AI 原生设计

**核心价值：**
- 像 Obsidian 一样本地优先、文件透明
- AI 和用户充分沟通后建设知识库
- 目录随业务调整而弹性变化
- 知识可积累、可追溯（双层页面 + 时间线）
- 混合搜索（语义 + 关键词 + 关系）

---

## 二、目标客户

| 维度 | 定义 |

## 三、核心设计原则

### 1. 技能不依赖目录

```
技能只依赖：内容类型(type) + 用户意图
目录 = AI和用户协商出来的共识，可调整
```

### 2. 目录弹性

```
Phase 1：先用通用结构
和AI充分沟通后，AI建议新目录 → 用户确认后才建
目录调整 → AI自动更新所有 frontmatter.type
```

### 3. 双层页面积累知识

```
页面只增不减，时间线 append-only
每次更新是叠加，不是覆盖
```

## 六、页面结构（双层设计）

### 上方：编译真相（当前状态）

```yaml
---
title: 页面标题
type: person          # person/company/project/meeting/deal/concept/research/collection
status: active        # active/archived
summary: 一句话摘要
confidence: low       # high/medium/low
tags: []
openThreads:
  - [ ] 待确认创始人背景
seeAlso:
  - [[相关页面A]]
relationships:
  - type: invested_in
    target: 红杉资本
    confidence: EXTRACTED  # EXTRACTED/INFERRED/AMBIGUOUS
    source: 腾讯新闻
created: 2026-04-29
updated: 2026-04-29
---

## 基本信息
- 待补充...

## Open Threads
- [ ] 待补充...

## See Also
- [[相关页面]]

---   <!-- 分界线，以下永不修改 -->

## 时间线（Append-only）

## [2026-04-29] 创建 | 页面初始化
```

### 核心字段

| 字段 | 必填 | 说明 |
|------|------|------|
| title | ✅ | 页面标题 |
| type | ✅ | 内容类型，决定存放位置 |
| status | ✅ | active=活跃，archived=归档 |
| summary | 建议 | 一句话摘要 |
| confidence | 建议 | 置信度 |
| tags | 建议 | 标签 |
| openThreads | 建议 | 待办事项 |
| seeAlso | 建议 | 关联页面 |
| relationships | 建议 | 关系抽取 |
| created | ✅ | 创建日期 |
| updated | ✅ | 更新日期 |

## 七、目录结构（弹性设计）

### Phase 1 最小结构

```
知识库/
├── index.md          # AI自动维护的内容目录（按 type 聚合）
├── log.md           # 操作日志（append-only）
│
├── .raw/            # 原始文件（Immutable）
│
└── 0-收集/          # 快速入口，不分类，等AI和用户协商后迁移
```

### 目录协商流程

```
新内容进入 0-收集/
    ↓
AI 分析内容类型 → 和用户协商"这是啥，放哪"
    ↓
用户确认 type
    ↓
如需新目录 → AI 建议 → 用户确认后才建
    ↓
文件迁移 + 更新 index.md
```

### Type 决定行为，不依赖目录

```typescript
// ❌ 旧：hardcode 目录
if (folder === '2-公司') { ... }

// ✅ 新：内容导向
if (frontmatter.type === 'company') { ... }
```

## 八、技能系统框架

### 目录结构

```
skills/
├── RESOLVER.md          # 🔑 决策树：收到内容 → 路由到哪个技能
├── enrich.md             # 入库：判断内容类型 → 和用户协商放哪
├── query.md              # 查询：搜索 + 综合回答
├── ingest.md             # 导入：处理外部文件/链接
├── maintain.md           # 维护：定期检查 + 提醒
└── conventions/
    ├── quality.md        # 质量标准
    └── citation.md       # 引用格式
```

### RESOLVER.md（不写死目录）

```markdown
# RESOLVER - 内容路由决策树

收到内容后，判断：

1. 是用户提问？
   → 走 query.md

2. 是外部文件/链接？
   → 走 ingest.md → enrich.md

3. 是待处理的新内容？
   → 走 enrich.md

enrich 判断逻辑：
- 有人名 → 问用户"这是谁？属于哪类？"
- 有公司名 → 问用户"这是公司还是项目？"
- 有会议/时间 → 建议进 4-会议/
- 缺信息 → 进 0-收集/，等用户确认后迁移

⚠️ 不写死目录！每次遇到新类型，和用户协商创建新目录。
```
